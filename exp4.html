1. External CSS

Styles defined in a separate .css file and linked to HTML document

üé® Global Styling

This entire page layout, header, navigation, and basic styling comes from the external CSS file styles.css.

/* In styles.css file */ body { font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif; background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); } .main-header { background: linear-gradient(135deg, #2c3e50 0%, #34495e 100%); color: white; padding: 40px 20px; }

üîó How to Link External CSS

External CSS is linked in the HTML head section using the link element:

2. Internal CSS

Styles defined within the HTML document using <style> tags

üéØ Page-Specific Styling

This green section and its styling is created using Internal CSS defined in the <style> tag within this HTML document's head section.

Internal CSS is perfect for unique styles that apply only to this specific page.

üé® Custom Design

Unique styles for specific pages without affecting others

‚ö° No Extra Files

All styles contained within the HTML document

üéØ Page-Specific

Styles that don't need to be shared across multiple pages

3. Inline CSS

Styles applied directly to HTML elements using the style attribute

INLINE CSS

üé® Direct Element Styling

This red section is styled entirely using Inline CSS. Each element has its styles defined directly in the style attribute.

‚ö° Immediate Effect

Changes apply instantly to the specific element

üéØ Highest Priority

Overrides both external and internal CSS

4. CSS Types Comparison

Understanding when to use each type of CSS

Aspect External CSS Internal CSS Inline CSS
Location Separate .css file <style> tag in <head> style attribute in element
Scope Multiple HTML pages Single HTML page Single HTML element
Priority Lowest priority (1) Medium priority (2) Highest priority (3)
Maintenance Easy - centralized Moderate - page level Difficult - element level
Reusability High - across pages Medium - within page Low - single element
File Size Cached by browser Adds to HTML size Increases HTML size
Best Use Site-wide styling Page-specific styles Quick fixes, testing

üí° Best Practice Recommendations:

  • External CSS: Use for consistent styling across multiple pages
  • Internal CSS: Use for page-specific styles and overrides
  • Inline CSS: Use sparingly for quick fixes or dynamic styling
